# Duologue 프로젝트 규칙

> 이 문서는 Claude가 코드를 작성할 때 **절대적으로 따라야 하는 규칙**입니다.

## [중요] 이모지 사용 금지

사용자가 명시적으로 요청하지 않는 한, 코드와 주석, 커밋 메시지에서 이모지를 절대 사용하지 않습니다.

## [중요] 패키지 매니저

**이 프로젝트는 Yarn만 사용합니다. npm 절대 사용 금지.**

- 사용: `yarn`, `yarn add`, `yarn remove`
- 금지: `npm`, `npm install`, `npm run`, `npx` (예외: husky init 등 일부 설정 명령어)

## 작업 프로세스

### 작업 관리 방식

**문서 기반 Task-Driven 개발** 사용

- `docs/tasks/active/`: 현재 진행 중인 작업
- `docs/tasks/pending/`: 향후 예정된 작업
- `docs/tasks/completed/`: 완료된 작업

### Warmup 필수 확인사항

1. `.claude-project-rules.md` 읽기
2. `docs/tasks/active/` 폴더 확인
3. `git status` 확인
4. 작업 컨텍스트 요약 제공

### TodoWrite 도구 사용

- 현재 세션의 임시 메모용만 사용
- 향후 작업은 `docs/tasks/pending/`에 마크다운 문서로 생성

### 작업 문서 작성 원칙

- 계획 수립 및 진행 상황 추적용 (실행 매뉴얼 아님)
- 핵심만 작성, 장황한 설명 금지
- 코드 예시는 필수 항목만 (SQL 스키마, 환경변수 등)
- 이미 코드/README에 있는 내용 반복 금지

## 코드 스타일 규칙

### TypeScript

1. **any 타입 사용 금지** - `unknown` 또는 명시적 타입 사용
2. **Strict 모드 준수** - `tsconfig.json`의 strict 옵션 활성화
3. **Path Alias 사용** - `@/components/...` (상대 경로 지양)

### React 컴포넌트

1. **Arrow Function으로 작성**
   ```typescript
   export const MyComponent: React.FC<Props> = ({ title }) => {
     return <View>{title}</View>;
   };
   ```

2. **Props 타입 명시**
   ```typescript
   interface ButtonProps {
     title: string;
     onPress: () => void;
   }
   ```

3. **Inline Text 금지** - 상수로 분리 (i18n 대비)

### Import 규칙

순서: builtin → external → internal → local (각 그룹 사이 빈 줄, 알파벳 순)

```typescript
import React, { useState } from 'react';
import { View } from 'react-native';

import { useQuery } from '@tanstack/react-query';

import { Button } from '@/components/ui/Button';

import { styles } from './styles';
```

### 변수 및 상수

- 미사용 변수 금지 (필요시 `_prefix`)
- 상수는 UPPER_CASE

## 스타일링 규칙

- **Inline Styles 금지** - `StyleSheet.create()` 사용
- **Tamagui 컴포넌트 우선** - 공통 컴포넌트 없을 때만 React Native 직접 사용

## 환경 변수 규칙

- `.env.local`, `.env.development`, `.env.production` - git 무시
- `.env.example` - git 커밋

## React Hooks 규칙

- 의존성 배열 정확히 명시
- Memoization 적절히 사용 (비용 큰 연산, 자식 전달 함수)

## 파일명 규칙

- 컴포넌트: PascalCase (`Button.tsx`)
- 훅: camelCase with use (`useAuth.ts`)
- 유틸/타입: camelCase
- 스토어: camelCase with Store (`authStore.ts`)

## 폴더 구조

```
duologue-app/
├── app/          # Expo Router 화면
├── api/          # API 클라이언트
├── components/   # 재사용 컴포넌트
├── hooks/        # 커스텀 훅
├── stores/       # Zustand 스토어
├── types/        # TypeScript 타입
├── utils/        # 유틸리티 함수
└── constants/    # 상수
```
