# Duologue 프로젝트 규칙

> 이 문서는 Claude가 코드를 작성할 때 **절대적으로 따라야 하는 규칙**입니다.
> 모든 작업에서 이 규칙을 최우선으로 준수해야 합니다.

## [중요] 이모지 사용 금지

**사용자가 명시적으로 요청하지 않는 한, 코드와 주석에서 이모지를 절대 사용하지 않습니다.**

```typescript

const handleLogin = () => { ... };
```

```typescript
// [X] 금지 - 커밋 메시지
git commit -m "feat: Add login feature"

git commit -m "feat: Add login feature"
```

## [중요] 패키지 매니저

### Yarn 사용 필수

**이 프로젝트는 npm을 사용하지 않습니다.**

- **사용**: `yarn`, `yarn add`, `yarn remove`, `yarn install`
- **절대 사용 금지**: `npm`, `npm install`, `npm run`, `npx`

### 명령어 예시

```bash
# [O] 올바른 방법
yarn install
yarn add package-name
yarn add -D package-name
yarn remove package-name
yarn start
yarn lint

# [X] 절대 사용하지 말 것
npm install
npm i package-name
npm run start
npx some-command  # 예외: husky init 등 일부 설정 명령어만 허용
```

## 작업 프로세스

### 작업 관리 방식 (중요!)

이 프로젝트는 **문서 기반 Task-Driven 개발**을 사용합니다.

#### 작업 문서 위치

- `docs/tasks/active/`: 현재 진행 중인 작업
- `docs/tasks/pending/`: 향후 예정된 작업
- `docs/tasks/completed/`: 완료된 작업 (아카이빙)

#### 세션 시작 시 필수 확인사항 (Warmup)

1. `.claude-project-rules.md` 읽기 (이 문서)
2. `docs/tasks/active/` 폴더 확인하여 현재 작업 파악
3. `git status` 확인하여 변경사항 파악
4. 작업 컨텍스트 요약 제공

#### TodoWrite 도구 사용 규칙

- **목적**: 현재 세션의 임시 메모 및 단기 작업 추적용만 사용
- **금지**: 실제 프로젝트 작업은 TodoWrite에 기록하지 않음
- **원칙**: 향후 작업은 반드시 `docs/tasks/pending/`에 마크다운 문서로 생성

#### 작업 문서 생성 규칙

- **새 작업 시작**: `docs/tasks/active/`에 문서 생성
- **작업 완료**: `docs/tasks/completed/`로 이동
- **향후 개선사항 발견**: `docs/tasks/pending/`에 새 문서 생성 (기존 문서 수정 금지)

#### 작업 문서 작성 원칙

- **목적**: 계획 수립 및 진행 상황 추적 (실행 매뉴얼 아님)
- **간결성**: 핵심만 작성, 장황한 설명 금지
- **코드 최소화**: 코드 예시는 필수 항목만 (SQL 스키마, 환경변수 등)
- **중복 제거**: 이미 코드/README에 있는 내용 반복 금지
- **명확성**: "무엇을" "왜" 하는지 중심으로 작성

## 코드 스타일 규칙

### TypeScript

1. **any 타입 사용 금지**

   ```typescript
   // [X] 금지
   const data: any = fetchData();

   // [O] 권장
   const data: UserData = fetchData();
   // 또는
   const data: unknown = fetchData();
   ```

2. **Strict 모드 준수**
   - `tsconfig.json`의 strict 옵션 항상 활성화
   - 타입 추론에 의존하되, 명시가 명확할 때는 타입 명시

3. **Path Alias 사용**

   ```typescript
   // [O] 권장
   import { Button } from '@/components/ui/Button';

   // [X] 지양
   import { Button } from '../../../components/ui/Button';
   ```

### React 컴포넌트

1. **함수형 컴포넌트는 Arrow Function으로 작성**

   ```typescript
   // [O] 권장
   export const MyComponent: React.FC<Props> = ({ title }) => {
     return <View>{title}</View>;
   };

   // [X] 금지
   export function MyComponent({ title }: Props) {
     return <View>{title}</View>;
   }
   ```

2. **Props 타입 명시**

   ```typescript
   // [O] 권장
   interface ButtonProps {
     title: string;
     onPress: () => void;
   }

   export const Button: React.FC<ButtonProps> = ({ title, onPress }) => {
     // ...
   };
   ```

3. **Inline Text 금지 (i18n 대비)**

   ```typescript
   // [X] 금지
   <Text>안녕하세요</Text>

   // [O] 권장 (상수로 분리)
   const TEXTS = {
     greeting: '안녕하세요',
   };

   <Text>{TEXTS.greeting}</Text>
   ```

### Import 규칙

1. **Import 순서**
   - builtin (React, React Native)
   - external (third-party packages)
   - internal (프로젝트 내부)
   - parent, sibling, index
   - 각 그룹 사이 빈 줄
   - 알파벳 순 정렬

   ```typescript
   // [O] 올바른 순서
   import React, { useState } from 'react';
   import { View, Text } from 'react-native';

   import { useQuery } from '@tanstack/react-query';
   import { create } from 'zustand';

   import { Button } from '@/components/ui/Button';
   import { useAuth } from '@/hooks/useAuth';

   import { styles } from './styles';
   ```

2. **미사용 Import 금지**
   - ESLint가 자동으로 제거
   - 커밋 전 확인 필수

### 변수 및 상수

1. **미사용 변수 금지**

   ```typescript
   // [X] 금지
   const unused = 'value';

   // [O] 필요시 언더스코어 prefix
   const _temp = 'value'; // 의도적으로 미사용
   ```

2. **상수는 UPPER_CASE**
   ```typescript
   // [O] 권장
   const MAX_RETRY_COUNT = 3;
   const API_TIMEOUT = 10000;
   ```

## React Hooks 규칙

### 의존성 배열 관리

```typescript
// [O] 권장 - 의존성 명시
useEffect(() => {
  fetchData(userId);
}, [userId]);

// [!] ESLint 경고 - 의존성 누락
useEffect(() => {
  fetchData(userId);
}, []); // userId가 의존성에 없음
```

### Memoization 적절히 사용

```typescript
// [O] 권장 - 비용이 큰 연산
const expensiveValue = useMemo(() => {
  return heavyCalculation(data);
}, [data]);

// [O] 권장 - 자식에 전달하는 함수
const handlePress = useCallback(() => {
  doSomething(id);
}, [id]);
```

## 파일 및 폴더 구조

### 파일명 규칙

- **컴포넌트**: PascalCase - `Button.tsx`, `UserProfile.tsx`
- **훅**: camelCase with use prefix - `useAuth.ts`, `useDebounce.ts`
- **유틸**: camelCase - `formatDate.ts`, `validation.ts`
- **타입**: camelCase - `user.ts`, `api.ts`
- **스토어**: camelCase with Store suffix - `authStore.ts`, `userStore.ts`

### 폴더 구조 준수

```
duologue-app/
├── app/          # Expo Router 화면
├── api/          # API 클라이언트
├── components/   # 재사용 컴포넌트
├── hooks/        # 커스텀 훅
├── stores/       # Zustand 스토어
├── types/        # TypeScript 타입
├── utils/        # 유틸리티 함수
└── constants/    # 상수
```
